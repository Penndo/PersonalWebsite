# 首页配置后端交互与数据流转（Mermaid）

本文档描述首页配置相关的端到端链路：前端 API 调用、后端 Controller/Service/ORM、数据库读写与返回。

## 1) 总体逻辑结构图

```mermaid
flowchart TD
    A[Homepage 页面组件] --> B[App useEffect 拉取首页配置]
    B --> C[src/services/api.ts axios 实例]
    C --> D[HTTP: localhost:4000]

    D --> E[NestJS main.ts + AppModule]
    E --> F[UserController /user/profile]
    E --> G[RecommendationController /recommendations]

    F --> H[UserService]
    G --> I[RecommendationService]

    H --> J[TypeORM Repository<UserProfile>]
    I --> K[TypeORM Repository<Recommendation>]

    J --> L[(MySQL/Postgres<br/>user_profile 表)]
    K --> M[(MySQL/Postgres<br/>recommendations 表)]

    L --> J --> H --> F --> C
    M --> K --> I --> G --> C
    C --> N[App setState]
    N --> O[Homepage 渲染 userInfo/recommendedItems]
```

## 2) `/user/profile` 数据流（查询 + 兜底创建）

```mermaid
sequenceDiagram
    participant FE as 前端 App.tsx
    participant API as api.ts userApi.getUserInfo()
    participant UC as UserController
    participant US as UserService
    participant UR as Repository<UserProfile>
    participant DB as DB:user_profile

    FE->>API: GET /user/profile
    API->>UC: HTTP 请求
    UC->>US: getProfile()
    US->>UR: findOne(where:id=1)
    UR->>DB: SELECT ... WHERE id=1
    DB-->>UR: 记录 or null
    UR-->>US: profile/null

    alt 未找到 profile
        US->>UR: create(defaultProfile)
        US->>UR: save(profile)
        UR->>DB: INSERT user_profile
        DB-->>UR: savedProfile
    end

    US-->>UC: UserInfoDto 映射结果
    UC-->>API: 200 JSON
    API-->>FE: res.data
    FE->>FE: normalize + setUserInfo
```

## 3) `/recommendations` 数据流（查询列表）

```mermaid
sequenceDiagram
    participant FE as 前端 App.tsx
    participant API as api.ts recommendationApi.getRecommendations()
    participant RC as RecommendationController
    participant RS as RecommendationService
    participant RR as Repository<Recommendation>
    participant DB as DB:recommendations

    FE->>API: GET /recommendations
    API->>RC: HTTP 请求
    RC->>RS: findAll()
    RS->>RR: find(order by order asc)
    RR->>DB: SELECT * ORDER BY order ASC
    DB-->>RR: rows
    RR-->>RS: entities[]
    RS-->>RC: entities[]
    RC-->>API: 200 JSON
    API->>API: map(itemId->id, itemType->type)
    API-->>FE: transformed list
    FE->>FE: filter(enabled) + sort + slice(0,4) + setRecommendedItems
```

## 4) `/recommendations/batch` 数据流（后台批量保存）

```mermaid
flowchart TD
    A[Admin 页面提交推荐项] --> B[recommendationApi.saveRecommendations]
    B --> C[转换前端结构 -> 后端结构]
    C --> D[POST /recommendations/batch]
    D --> E[RecommendationController.saveMultiple]
    E --> F[RecommendationService.saveMultiple]
    F --> G[deleteAll -> repository.clear]
    G --> H[(recommendations 全表清空)]
    H --> I[map 每项并重排 order=index+1]
    I --> J[repository.save 批量写入]
    J --> K[(INSERT 多条 recommendations)]
    K --> L[返回最新推荐列表]
```

## 5) 关键实现文件索引

- 前端调用入口：`src/App.tsx`、`src/services/api.ts`
- 后端启动与数据库连接：`backend/src/main.ts`、`backend/src/app.module.ts`
- 用户信息链路：`backend/src/user/user.controller.ts`、`backend/src/user/user.service.ts`、`backend/src/user/user-profile.entity.ts`
- 推荐项链路：`backend/src/recommendations/recommendation.controller.ts`、`backend/src/recommendations/recommendation.service.ts`、`backend/src/recommendations/recommendation.entity.ts`
